Overview

Timezone comparison, built from scratch

TZComp started as a WordPress Gutenberg block and was ported to React Native as an exercise in cross-platform development. The app runs on Android and web from a single codebase, with platform-specific behaviour handled via React Native's Platform API where needed. Every interactive component — dropdowns, number inputs, colour panels — is custom-built with no third-party UI library.

One codebase, three targets. The same timezone logic, theme system, and component library runs on Android, iOS, and web.

How it works

Offset-anchored comparison

Comparison is anchored to a shared UTC reference time. When the user adjusts any card to a hypothetical local time, the delta is applied to the underlying UTC timestamp and propagates automatically through every card, keeping all timezones consistent relative to the chosen moment. DST transitions are handled automatically — each card recalculates its UTC offset when the timezone transitions between standard and daylight time.

Features

What's under the hood

🌍
Dynamic card comparison
Add and remove timezone cards at will. Each card displays live time, date, timezone badge, and full timezone name, updating in sync with the UTC reference time.
🕐
Local time selection
Select a hypothetical local time on any card to see how it translates across all other timezones simultaneously — useful for scheduling across regions.
🎨
Runtime theme switching
A custom theme system derives all colours from four base values using a JavaScript implementation of CSS color-mix(). Preset palettes and fully customisable gradient accents switch at runtime without reloading.
🔢
Custom input components
Searchable dropdowns with Modal-based layering, adaptive opening direction and height based on available screen space, keyboard-aware repositioning, and number up/down inputs with inline text editing — all built without third-party UI libraries.

Technical decisions

Why these choices

Custom components
Dropdowns, number inputs, and colour panels are all custom-built — partly because the WordPress originals were custom, and partly because React Native's built-in equivalents are either limited or platform-inconsistent.
UTC-anchored time state
Rather than deriving time from a base card, all cards compute their local time from a shared UTC timestamp. This ensures AM/PM toggles, DST transitions, and day rollovers are always consistent across cards without any card-to-card synchronisation logic.
Intl.DateTimeFormat
Timezone offset calculations use Intl.DateTimeFormat exclusively rather than Date.toLocaleString, which behaves inconsistently on Android's Hermes engine. This ensures correct offset values across all platforms.
React context for theme
Theme and settings are managed via separate context providers, allowing any component to consume the current accent colour or user preferences without prop drilling. All derived colours are computed at context level and passed down as a flat theme object.
Modal-based dropdowns
Dropdowns render their content in a React Native Modal, positioned via measureInWindow, to escape the stacking context limitations that make z-index unreliable in deeply nested component trees on Android.
Expo SDK 52
Targeting SDK 52 rather than the latest release due to a known Gradle compatibility issue with SDK 53+ on Windows. An upgrade path to SDK 56 is planned once upstream toolchain issues are resolved.
