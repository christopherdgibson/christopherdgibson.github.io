Overview

Automated document collection, across multiple sources

The Report Download Hub aggregates publicly available documents from three institutional sources simultaneously. Configure filters for each source independently — document type, keyword, year or date range — check the sources you want, and submit. Matching PDFs are downloaded directly to your chosen directory, with options to skip existing files, append timestamps, and maintain a versioned history folder.

The user's machine is the server — no infrastructure, no authentication handoff, no data leaving the local environment.

Supported Sources

Three sources, one submission

Each source is independently configurable and optional. Sources can be combined freely — the output is the union of matching documents across all selected sources. The architecture is extensible to additional sources, including internal sites requiring authentication.

Features

What's under the hood

🏭
Model factory architecture
A factory pattern resolves the correct service model per source. Each service inherits from a typed base — EUStudiesService : BaseService<EUStudiesModel> — with source-specific navigation and download logic as needed.
⚡
Element-driven automation
No arbitrary delays. Selenium waits for specific DOM elements before proceeding, keeping automation fast and reliable without hardcoded timeouts.
📦
Smart download options
Skip files that already exist, append download timestamps, and optionally maintain a History folder with versioned copies. Confirmation prompt if the batch exceeds a configurable threshold.
🔍
Extended keyword search
Title keyword filtering beyond what the native site search exposes — applied as a post-retrieval filter across result sets from any source.
💻
Cross-platform desktop
Built with Avalonia UI — runs on Windows, macOS, and Linux without modification. The user's machine provides the browser and filesystem; no server required.
🔌
Extensible by design
Adding a new source means implementing the base interface — existing services and the download pipeline require no modification. Internal sources requiring authentication are supported by the same pattern.

Technical decisions

Why these choices

Avalonia UI
Cross-platform desktop framework for .NET — runs on Windows, macOS, and Linux without platform-specific code.
Selenium
Full browser automation is required — JavaScript-heavy sites lose all state on navigation, ruling out headless HTTP approaches.
Model factory
Decouples source resolution from download logic. New sources are added by implementing an interface, not by modifying existing code.
Desktop over web
Browser automation requires a persistent process with filesystem access — a desktop app distributes this to the user's own machine cleanly.
Element waits
Waiting for specific DOM elements rather than fixed delays makes automation faster and more robust across varying network conditions.
