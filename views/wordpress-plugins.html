Overview

Custom Gutenberg blocks, built from scratch

Both plugins are built as custom WordPress Gutenberg blocks using React, with no reliance on WordPress-specific block APIs — the editor and frontend interfaces are written directly, giving full control over both the authoring experience and the rendered output. The block editor layer is designed for intermediate WordPress designers: click to add a card, click to split it, click to add a description. No shortcodes, no custom admin panels.

Two interfaces per plugin — one for the designer building the page, one for the visitor reading it. Both written from scratch.

The Plugins

Two blocks, two problems

Plugin 01
Daily Feed
A Gutenberg block that fetches and displays a daily fact or quote from an external API, navigable by date via an integrated calendar. API requests are routed through a WordPress PHP proxy to avoid CORS restrictions, with a retry mechanism that handles timeouts and malformed responses. The block normalises both JSON and raw HTML API responses into a consistent display format.
Plugin 02
Agenda Block
A structured content card block for presenting items with expandable detail — originally designed for weekly schedules but general enough for any structured content. Cards can be split into sub-items (morning/afternoon sessions, for example), each generating its own description button automatically. On desktop the detail panel opens to the right of the card list; on mobile it overlays. All layout logic is custom.

Features

What's under the hood

Daily Feed

🔄
CORS proxy with retry
API requests route through a PHP WordPress proxy when direct fetch would be blocked. A retry function handles cURL timeouts and network failures with configurable attempts and delay, failing gracefully rather than silently.
📅
Calendar navigation
Daily Feed includes a date-picker calendar that updates the displayed content on selection. Each date fetches its own API response, with the result cached to avoid redundant requests on revisit.

Agenda Block

🃏
Dynamic sub-card generation
Splitting a card in the Agenda Block automatically generates the corresponding description buttons — no manual wiring required. The number of sub-items per card is unconstrained.
📱
Responsive detail panel
The expanded description panel sits to the right of the card list on desktop and overlays on mobile — layout logic written entirely without a CSS framework.
✏️
Designed for designers
The block editor interface exposes only what a WordPress designer needs — text fields, add/split/remove controls — with no code exposure. The save output is clean and preview-ready.

Technical decisions

Why these choices

React
Gutenberg's native block development framework — used directly rather than through abstraction layers to keep full control over block behaviour.
PHP proxy
Server-side request routing through WordPress admin-ajax.php bypasses browser CORS restrictions without requiring a separate backend service.
No WordPress APIs
Custom implementations rather than WordPress block data store — the logic is more explicit, fully owned, and not dependent on WordPress API stability.
XAMPP / Apache
Local development environment with Apache server and MySQL — a full WordPress stack running locally for development and testing.
Response normalisation
Daily Feed handles both JSON and raw HTML API responses, normalising them into a consistent display format rather than assuming a fixed structure.
