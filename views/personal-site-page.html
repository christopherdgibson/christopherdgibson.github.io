Overview

Built from scratch — no frameworks, no templates

This site is itself a portfolio piece. Every component — the responsive navbar, the animated hamburger sweep, the SPA routing system, the mobile overlays — is custom-coded in vanilla HTML, CSS, and JavaScript without any framework or library. The decision to avoid frameworks was deliberate: the site demonstrates what can be built with the fundamentals alone, and keeps the codebase fully owned and understood.

No framework to hide behind — every behaviour is written, understood, and intentional.

Features

What's under the hood

✦
What is a hamburger? Click to find out! →Hamburger sweep animation
On mobile, opening the nav wipes a light overlay across the header name. Each letter transitions to the accent blue with a staggered delay — right to left on open, left to right on close — achieved with per-character span injection and CSS transitions.
🗂️
SPA routing
Pages load dynamically without full reloads. A view callback system runs page-specific initialisation functions after each navigation — anchor buttons, SVG icons, feature card overlays, footer buttons, and screenshot carousels are all initialised on demand.
📱
Responsive at every width
The navbar collapses to a slide-out vertical menu on mobile. The sub-menus expand over content on desktop and slide-out from alternating directions on mobile. Feature cards pop out with a dimmed background overlay on mobile hover, resolving the ambiguity between decorative borders and clickable links.
🖼️
Screenshot carousel
Project pages include a scrollable screenshot carousel with left/right chevron navigation. Screenshots can be viewed inline in the hero or expanded to a full overlay. Images are cropped consistently using object-fit and object-position to maintain a uniform hero height across varying screenshot dimensions.
⚡
SVG performance strategy
SVG icons are loaded from separate files to keep markup clean, with deliberate exceptions for performance — persistent header, footer, and scroll-to-top SVGs are hard-coded inline on the index page to eliminate flash on initial load for above-the-fold elements.
🎬
Curtain navigation
When navigating from a sub-menu item in mobile, the main menu slides right and the sub-menu slides left simultaneously, returning to home position, as the new page loads — a curtain-opening effect that registers subconsciously without demanding attention.

Technical decisions

Why these choices

No framework
Demonstrates command of fundamentals. Every behaviour is written from first principles — nothing is delegated to a library whose internals are opaque.
SPA routing
Dynamic page loading without reloads preserves header and footer state, enables transition animations, and avoids the flicker of full page navigation.
SVG strategy
Lazy-loaded SVGs for non-critical icons keep markup clean. Hard-coded inline SVGs for above-the-fold elements eliminate flash on first load.
Base callbacks
A shared initialisation system runs on every page load. Each function checks for its target elements before running — pages that don't need a feature pay only a null check.
Mobile overlays
Feature card hover borders read as links on touch devices, so a pop-out overlay with dimmed background resolves the link/decoration ambiguity without changing the desktop experience.
Declarative navigation
Footer back/next buttons are configured via hidden data-attributes on each page and read dynamically on load — each page owns its navigation content while events handlers are assigned centrally.
