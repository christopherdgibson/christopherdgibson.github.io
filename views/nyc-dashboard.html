Overview

Explore NYC 311 data, live

NYC 311 is the non-emergency government information and services call center for residents of New York City. This dashboard pulls from the NYC Open Data API and lets you filter service requests across all five boroughs, individual precincts, and zip codes. Every selection — adding a borough, removing a zip code — propagates instantly across all charts and tables on the page without a reload. The colour palette draws from themes of the city itself: the primary blue of the NYC flag, the accent yellow of the NYC 311 logo.

Built to be shared freely — no server, no hosting costs, no maintenance. Just a URL.

Features

What's under the hood

⚡
Live state updates
Selections propagate immediately to all charts and tables. Tight state dependency with no unnecessary re-renders.
🏗️
Clean service architecture
Pages interact only with RequestService, ChartService, and LayoutService. JS interop is isolated to LayoutService alone.
📦
Fully custom components
Every component is encapsulated with no knowledge of the rest of the solution — sortable tables, expandable text, loading overlays, slide-out sidebars.
📊
ApexCharts via typed interop
Chart configuration is strongly typed in C# and passed to ApexCharts through a JS interop layer, keeping rendering concerns cleanly separated.
📱
Responsive at every width
Charts and tables wrap to available space. Navbar collapses to a slide-out overlay on mobile. Sidebar shifts content on desktop, overlays on mobile.
📄
PDF export
Full page export via html2pdf with custom CSS rules to prevent charts and tables from being cut mid-element across page breaks.

Technical decisions

Why these choices

Blazor WebAssembly
Compiles to static files, enabling free hosting on GitHub Pages with no server required.
NYC Open Data API
Publicly available, rich enough to make filtering and aggregation genuinely interesting.
ApexCharts
Mature JS charting library integrated via a typed C# interop layer rather than manual canvas rendering.
Service isolation
JS interop lives in one place. Pages never touch the DOM directly. Messaging and loading state flow through services.
html2pdf
Free, client-side PDF generation without server processing or third-party services.
