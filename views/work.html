Approach

How I work

🏗️
Architecture first
Every project starts with a deliberate structural decision — how concerns are separated, what depends on what, and why. The code follows from the design.
🔍
Motivated by constraints
Static hosting drove the Blazor choice. Browser automation drove the desktop app. The constraints of each problem shaped the solution rather than being worked around.
⚡
Detail without excess
Sortable columns, expandable cells, responsive sidebars, bi-directional animations — details that serve the user without drawing attention to themselves.
🌍
Domain-aware development
A background in economics and tax informs what gets built and why. The tools are chosen to solve real problems in domains I understand from the inside.
